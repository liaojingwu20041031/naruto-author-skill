# ✔️ 交付级质量检查清单 (Quality Checklist)

在 `Naruto Author AI` 创作流水线中，无论是大纲、人物卡还是单章正文，在最终交给读者（或发布）之前，必须由 **主笔作家** 和 **DeepSeek 暗部审核官** 联合进行这最后一道**六维质量质检 (6-Dimensional Quality Check)**。主笔可由 Claude Code / Codex 或其他本地代码代理承担；4000-5000 字长章的完整毒点审核优先交给 DeepSeek，避免本地代理超时。

---

## 第一维度：Character Core (角色根子检验) 🛡️

- [ ] **口头禅适度**：少量原著口头禅、称呼和语气用于角色辨识或喜剧点；没有机械堆砌罗马音/日式尾巴。
- [ ] **梗从处境里长出**：制度梗、误会梗、人物关系梗服务场景，没有打断生死场面或让全员变成现代嘴替。
- [ ] **动机连贯性**：主角或原著配角的行为选择，符合 `character-sheet.md` 中设定的“深层动机”，没有为了推动剧情而强行降智或变成纯粹的工具人。

## 第二维度：Lore & Power Scaling (设定与战力核对) ⚔️

- [ ] **物理底线**：结印、查克拉量消耗、属性克制原则**没有被违背**。
- [ ] **金手指/外挂**：主角没有突然爆发出超纲的能力。如果使用了外挂，**明确描写了代价或冷却限制**。
- [ ] **跨体系融合**：引入的新能力体系（如果有）能分步接进忍界规则，没有强行降维打击或无代价硬覆盖。

## 第三维度：Narrative Focus (主线与节奏审查) 📈

- [ ] **有效内容**：本章留下了至少一项有效内容：主线推进、关系变化、世界观信息、伏笔、角色记忆点、笑点或未来蝴蝶效应种子。
- [ ] **日常不空转**：吃饭、赶路、洗澡、抓猫等日常如果出现，承担了关系、伏笔、能力暴露、制度梗或角色塑造功能。

## 第四维度：Show, Don't Tell (画面感与信息展现) 🎬

- [ ] **没有设定说明文**：没有超过三句话的、像教科书一样的纯忍术原理解释或历史背景旁白。
- [ ] **战斗细节化**：战斗不是“他用了一个火遁，敌人死了”，而是写出了结印的动作、查克拉涌动的体感、忍术交锋的环境破坏、以及双方战术博弈的心理战。
- [ ] **对话有动作**：对话不只是两个名字在轮流说话，而是配有表情、小动作或环境互动。

## 第五维度：The Hook (致命悬念与读者粘性) 🪝

- [ ] **章末钩子有效**：本章结尾**不是**平淡的收尾，而是运用了 `hook-techniques.md` 中的 10 种悬念技巧之一（如：诡异的宁静、面具碎裂、倒计时开启等）。
- [ ] **断章位置合法**：绝对没有在一句话说一半、或一个忍术刚结印一半时强行断章（太监断章）。
- [ ] **小剧场互动 (可选)**：在正文区之外的 `📝 作者的话` 区域，通过作者的口吻（或主角的脱线吐槽）埋下了互动悬念或求票萌点。

## 第六维度：Emotional Arc (情绪弧线与过渡) 🌊

- [ ] **事件驱动情绪**：能标出“事件 -> 可见反应 -> 行动选择 -> 余波”，情绪来自角色经历而不是旁白宣布。
- [ ] **Show Don't Tell**：没有用“愤怒、绝望、不甘在胸腔翻滚”“无力感扼住咽喉”等抽象句式替代体验。
- [ ] **情绪不空转**：没有同一情绪无新事件支撑地反复换词重说，也没有让抽象心理独白压过事件推进。
- [ ] **轻小说节奏合格**：情绪落点后推进下一个事件、动作、对话、发现或关系变化。
- [ ] **字数达标**：正文主体达到 4000-5000 字；扩写内容用于事件链、互动、画面和行动推进，不是设定水文或纯心理抒情。

> ⚠️ **暗部审核分级**：
