# 📜 章节生产标准模板 (Chapter Output Template)

本模板是主笔作家（Claude Code / Codex 均可）在生成单章正文时**必须严格遵守**的输出格式。
它集成了小说正文、颜文字/梗的隔离区（作家专区）、以及用于对接暗部审核的元数据。

---

## 模板结构要求

在生成正文时，必须按照以下 Markdown 格式输出：

```markdown
# 第 {章回号} 章：{极具张力且概括本章核心悬念的四个字/八个字标题，如：血月之夜 / 撕裂的伪装}

**【当前时间线】**：木叶X年 / 九尾之乱当晚
**【本章视角】**：{主角名字 / 或配角名字}

---

{这里是 4000-5000 字的小说正文内容}

{正文要求：}
{- 严格遵循“画面感叙事”，大段动作描写、忍术对轰的细节拆解。}
{- 战斗中加入查克拉流动的微观感知。}
{- 本书是轻松快乐向、二创向、系统文：允许制度梗、误会梗、系统奖励反差、角色可爱的风格化偏移和少量原著口头禅。}
{- 严禁大段设定说明；设定应通过对话、误会、战斗反馈、任务文书、暗部报告或系统提示自然带出。}
{- 遵循轻量情绪弧线：事件 -> 可见反应 -> 行动选择 -> 余波。不要机械套公式。}
{- 情绪必须由动作、物件、环境、对话潜台词承载；避免同一情绪无新事件支撑地反复换词重说。}
{- 扩写 4000-5000 字时优先扩写事件链、人物互动、制度梗、误会滚雪球和场面调度，不扩写纯心理独白。}
{- 搞笑吐槽可以缓冲压力，但不能让人物压力清零；笑点要从人物处境里长出来。}
{- 严禁在极其严肃的剧情中出现颜文字，只有在日常对话或心理吐槽中才允许自然植入 ಠ_ಠ 这种符号。}
{- 确保本章结尾落在《忍界悬念设置技巧》的某个强力钩子（Hook）上。}

---

### 📝 作者的话 (Author's Note / 小剧场)
{此处为绝对安全区！你可以尽情在这里：}
{- 发挥二次元属性，适度玩梗。}
{- 用极其轻松、沙雕的语气调侃本章主角的惨状或配角的槽点。}
{- 大量使用颜文字 (╯°□°）╯︵ ┻━┻ 或现代流行词。}
{示例：卡殿今天又迟到了呢 (ಡωಡ)！大家猜猜下一章带土会不会被主角按在地上摩擦？求推荐票求打赏呀～！}

---

### 🛡️ 暗部查验数据 (用于审核代理读取)
- **核心推动**：[主线 / 羁绊 / 蝴蝶效应]（简述本章推进了什么）
- **出场人物**：[列出本章出场的关键人物]
- **战力变动**：[本章消耗了什么 / 获得了什么成长]
- **情绪弧线**：[起点 -> 触发 -> 缓冲 -> 行动 -> 落点，并说明下一章要承接的情绪余波]
- **章末钩子类型**：[如：面具碎裂 / 查克拉枯竭 / 诡异的宁静]
```

## 注意事项：
1. **正文与元数据的绝对隔离**：正文区域绝对不能出现“[此处应有战斗描写]”这种 AI 过程思考痕迹。正文必须是可直接发表的纯净版本。
2. **作者的话**：这是拉近与读者距离的同人圈核心文化。可以在这里求票、吐槽、抛出悬念互动，增强追更粘性。
